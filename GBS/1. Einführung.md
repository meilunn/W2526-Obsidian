## Offenes System
- Besteht aus ==Komponenten== 
- Verbingungen zwischen Komponenten beschreiben ==Abhängigkeiten== 
- Besitzt ==Schnittstellen== zur externen Einwirkung auf das Sys
- so gestaltet und dokumentiert, dass die ==Schnittstellen== von Dritten genutzt werden können
### Schnittstelle
- z.B. Benutzerinteraktion, Netzschnittstelle
- ermöglichen verschiedene Sichten auf ein Sys
- Von außen: Black-Box-Sicht
	- Zusammenfassungen von Komponenten und Abgrenzungen zum Nutzer
- Von innen: White-Box-Sicht
	- Rekursive Aufteilung in Komponenten bzw. Sub-Komponenten

# Prozesse
## Definition
- Prozess hat Stack, Daten, Code
- BS sorgt dafür dass Prozesse sich unabhängig davon bewegen können
	- jeder Prozess bekommt die Illusion
		- die gesamte CPU/RAM für sich zu haben
		- einen eigenen Speicher von 0x000000 bis 0xFFFFFF -> Virtueller Adressraum ist Abstraktionen des physischen Speichers durch BS
			- Abbildung im System
			- eigentlich sind diese Prozesse an unterscheidlichen Stellen im tatsächlichen Speicher
			- nicht möglich für Programm außerhalb von seinem zugeteilten Bereich darauf zuzugreifen
	- BS teilt ein, wer wann laufen darf -> Ressourcenzuweisung
	- bevor CPU einem P weggenommen wird wird ein snapshot gemacht und hinterlegt
		- wenn P wieder drankommt kann der Zustand wiederhergestellt werden
