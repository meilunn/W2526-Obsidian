> Zugriff
- Last in - first out
- `rsp` - Stack Pointer

> [!warning] Stack wächst von großer Adresse zu kleine Adressen
- push/pop reg64/mem64
- `push op` 
	- `sub rsp, 8; [rsp] <- op`
	- rsp um 8 verringern ist weiter nach unten im Stack
- `pop op`
	- `op <- [rsp]; add rsp, 8
	- rsp + 8 ist zurück nach oben im Stack

## Funktionsaufrufe
- Funktionsaufruf impliziert Speicherzugriff auf den Stack
- `call label/reg64`
	- in reg64 ist ein Fktpointer gespeichert
	- -> "push rip"; jmp op
		- pusht den jetztigen Wert von rip (-> Rücksprungadresse) auf Stack und springt zum Fkt
- `ret`
	- "pop rip"
