# Disassembling
- `objdump <prog_name> -d -M intel | less`
- ![[Pasted image 20251112173742.png|500]]
	- 1. Spalte: Ort der Befehle in Byte
	- 2. Spalte: Inhalte der Binärdatei
		- Befehl in Maschienencode
		- viele Befehle größer als 1 Byte, weshalb die Angaben in der 1. Spalte springen
	- 3. Spalte: generierter Assemblercode
	- `leave` = mov rsp, rbp; pop rbp

# Optimierungsstufen
- `-O0` Keine Optimierung (default)
- `-O1`, `-O2`, `-O3` 
- `-Ofast`: O3 + Floating-Point Optimierungen -> evtl Rungdungsfehler
- `-Os`: Möglichst kleine Ausgabedatei
- `-Og`: Gut debugbarer Code
